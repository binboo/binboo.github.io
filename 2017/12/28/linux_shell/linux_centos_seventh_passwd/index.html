用户和用户组管理

发表于 2017-12-28 | 分类于 IT技术

用户管理简介：

1、越是对安全性能要求高的服务器，越需要建立合理的用户权限等级制度和服务器操作规范。
2、在Linux中主要是通过用户配置文件来查看和修改用户信息的。

用户配置文件

用户信息文件 /etc/passwd

要查看配置文件passwd的帮助信息，可用如下命令：

如果不加5，默认为1，即查看命令passed的帮助信息。

  • 第一字段:用户名称
  • 第二字段：密码标识x，说明该用户有密码，没有标识表示该用户没有密码。
  • 第三字段：UID
    0 超级用户；1-499 系统用户（伪用户）；500-65535：普通用户。
    如果某用户UID为0，则它为超级用户。
  • 第四字段：GID（用户初始组ID）
    初始组：用户一创建的默认组，每个用户的初始组只有一个，一般就是和这个用户用户名相同的组就是初始组。
    附加组：指用户可以加入多个其它组，并且拥有这些组的权限，附加组可以有多个。
  • 第五字段：用户说明
  • 第六字段：用户家目录
    普通用户：/home/用户名
    超级用户：/root/
  • 第七字段：登陆后的shell
    命令解释器是实现交互的基础。
    shell是Linux的命令解释器。
    输入命令，命令解释器解释传输给内核，内核再作出应答响应。在/etc/passwd中，除了标准的shell是/bin/bash外，还可以写成/sbin/nologin表示禁止登陆的用户。

初期用户名密码以密文方式存放在passwd中，但是由于passwd拥有的权限较高，后来将密码放在了/etc/shadow文件中，shadow的权限为000。

只有本地登陆允许用户没有密码，对于用ssh进行的远程登录，必须要求有用户密码。

密码原则：易记性，复杂性，时效性

影子文件 /etc/shadow

文件权限：000

  • 第一字段：用户名
  • 第二字段：加密密码
    注意：由于加密算法为SHA512散列的加密密码，相同的密码会换算出不同的密码串。如果密码为”!!”或者”*”，表示用户没有密码，不能登陆。
  • 第三字段：密码最后一次修改时间
    使用1970年1月1日作为标准时间，每过一天时间戳加一。
  • 第四字段：两次密码的修改间隔，比如时间间隔为10，表示上一次修改密码后过10天才能再次修改密码。
  • 第五字段：密码有效期（99999永久生效）
  • 第六字段：密码修改到期前的警告天数（和第五字段相比）
  • 第七字段：密码过期后的宽限时间（和第五字段相比）
    0 代表密码过期后立即失效
    -1 代表密码永远不会失效
  • 第八字段：账号失效时间，用时间戳表示（优先级比密码有效期高）
  • 第九字段：保留字段

时间戳换算时间 ：data -d “1970-01-01 17476 days”

组信息文件： /etc/group (与用户信息文件/etc/passwd对应)

  • 第一字段：组名
  • 第二字段：组密码标志
  • 第三字段：GID
  • 第四字段：组中附加用户

组密码文件 /etc/gshadow

  • 第一字段：组名
  • 第二字段：组密码
  • 第三字段：组管理员用户名
  • 第四字段：组中的附加用户

用户管理相关文件

用户的家目录

普通用户：/home/用户名/，所有者和所属组均是此用户，权限为700.
超级用户：/root/，所有者和所属组都是root用户，权限是550.

Linux中通过提示符判断用户是否为root用户，$为普通用户，#为超级用户。

用户的邮箱

/var/spool/mail/用户名

passwd命令有如下几个常用选项：

  • -S 查看密码信息
  • -l 锁定用户 实质：/etc/shadow下的密码前加入“!!”
  • -u 解锁用户
  • –stdin 可以通过管道输出的数据作为用户的密码。

修改用户信息命令usermod

usermod选项如下：

  • -u UID:修改用户的UID号。
  • -c 用户说明：修改用户的说明信息。
  • -G 组名：修改用户的附加组。
  • -L 临时锁定用户
  • -U 解除用户锁定
