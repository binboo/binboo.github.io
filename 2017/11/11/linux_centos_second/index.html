权限管理命令chmod

chmod命令是Linux中最重要的权限管理命令，文件所有者与root用户均可运用chmod改变权限。chmod的英文原意为change the permissions mode of a file
该命令所在路径为\bin\chmod.chmod有两种不同的语法方式，下面分别加以介绍：

chmod第一种语法规则

chmod [{ugoa}{+-=}{rwx}][文件或目录]

  • ugoa分别代表文件所有者（user）,组所有者（group）,其他用户（others）
  • +-=分别代表增加权限，减少权限，权限重置。
  • rwx分别读权限，写权限，执行权限。

例如下面的一条命令的含义为将当先目录的boy文件赋予执行权限。

chmod的-R参数

chmod的-R参数是递归修改，当加入-R参数时，当前修改的文件夹以及文件夹中的所有文件的权限同时被修改。

注意：文件夹与文件的rwx所代表的含义并不相同，文件的r权限代表可以查看文件内容，w权限代表可以修改文件内容，x权限代表可以执行文件。但文件夹的r权限代表可以列出目录中的内容，w权限代表可以在目录中创建、删除和修改文件，x权限代表可以进入目录。因此，某个文件没有写权限但其所在文件夹有写权限时，该文件依然可以被删除。
由上面的定义也可以看出，文件夹的r与x权限一般是成对存在的，很少有文件夹只有r权限没有x权限或只有x权限没有r权限。

权限管理命令umask

如果单独执行umask不加任何参数，其值为0022，解释如下：

  • 第一位0表示特殊权限
  • 后三位022为权限掩码，其真实权限为777-掩码权限由于默认为022，故目录真实权限为755即drwxr-xr-x
