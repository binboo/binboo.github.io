利用HEXO建立个人静态网站(一)

发表于 2017-11-08 | 分类于 IT技术

由于HEXO的快速强大的功能受到人们广泛的关注，前两天我利用课余时间搭建了自己的个人博客网站，我采用HEXO搭建个人网站主要是基于以下几点考虑：

  • 个人博客用户访问量不大，没有太多功能方面的要求，采用没有数据库支持的静态博客完全可行

  • HEXO支持轻量级的MARKDOWN语法。

  • 符合开源精神，自主操控性较强，是一个自由、安静、个性化的写作平台。

下面我就搭建过程中的主要步骤以及搭建过程中遇到的错误做一些说明（以下内容全部基于UBUNTU 16.04）：

安装nodejs

简单说明一下，apt和apt-get没什么大区别，apt有逐渐替代apt-get的趋势。

我们安装的npm是Node.js的包管理工具（node.js package manager），我们为何需要一个包管理工具呢？因为我们在Node.js上开发时，会用到很多别人写的JavaScript代码，如果我们要使用别人写的某个包，每次都根据名称搜索一下官方网站，下载代码，解压，再使用，非常繁琐。于是一个集中管理的工具应运而生：大家都把自己开发的模块打包后放到npm官网上，如果要使用，直接通过npm安装就可以直接用，不用管代码存在哪，应该从哪下载。

更重要的是，如果我们要使用模块A，而模块A又依赖于模块B，模块B又依赖于模块X和模块Y，npm可以根据依赖关系，把所有依赖的包都下载下来并管理起来。否则，靠我们自己手动管理，肯定既麻烦又容易出错。
