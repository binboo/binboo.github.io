find命令有很多参数可以选择，这里介绍如下几个参数

  • -a -o 条件满足查找
  • -type 根据文件类型查找
  • -inum 根据i节点查找
  • -exec/-ok 对操作结果执行操作

这里介绍通配符的概念，来匹配模糊搜索：

  • 通配符 * 表示匹配任意多个字符（包括零个）
  • 通配符 ? 表示匹配单个字符。

因此，像*init*表示文件名中包含init的文件，init*表示以init开头的文件。

但这里还有一个问题，由于Linux严格区分大小写字母，但有的时候我们并不希望查找时区分大小写，因此linux的find命令提供了一个-inmae选项，这样find就不再区分大小写了(Linux的许多查找命令中-i均为不区分大小写的选项)。

  • +n表示查找大于指定大小n的文件

这个例子的含义是查找/etc下大于100M的文件，大家可能会疑问为什么是100M，不应该是200M吗，要解释清楚这个问题，我们需要明白数据块的概念：

  • 数据块是find大小查找命令中的基本单元，一个数据据块为512字节，也即0.5KB，因此我们在算出文件是多少KB之后再乘2得到的大小才能写到-size后面。

100MB=102400KB=204800数据块，因此命令中写的是204800。

-a -o

-a为and的缩写，该属性表示前后的条件均满足
-o为or的缩写，该属性表示前后条件满足一个即可

-type

该选项意为按照文件类型查找，后面可以跟

  • f 查找类型为 文件
  • d 查找类型为 目录
  • l 查找类型为 软链接文件

-inum

表示按照i节点查找

该查找选项的作用通常有以下两个：

  • 删除一些文件名复杂的文件
  • 查找某文件是否存在硬链接

-ok选项与-exec选项的区别在于：-ok选项对每一个查找到的文件执行询问操作，y则执行，n则不执行。而-exec则没有这一询问，对所有查找到的文件直接执行后面的命令。因此，对于一些涉及删除的重要操作，用-ok命令可以保证万无一失。

命令格式：

搜索原理：locate搜索命令的机理与find不同，locate不在整个硬盘中搜索文件，而是建立了一个定期更新的文件资料库，找文件时候直接在资料库中寻找。

优势：搜索速度较find快很多（类似于win上的everything）

劣势：对于新建立的文件并没有更新到资料库中，因此无法找到新建立的文件。

适用场景：locate命令最好的应用场合就是搜索一些长期存在的系统文件。

locate手动更新资料库命令

当我们新建立一个文件时locate并不能搜索到它，但如果我们手动更新资料库，我们就能用locate搜索到新建文件了。

locate也有-i选项，意为不区分大小写搜索。

该命令格式与which相同，结果显示命令的绝对路径，但不显示命令的别名，取而代之的是关于该命令的帮助文档。
